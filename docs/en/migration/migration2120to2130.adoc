=== coffee-module-mongodb

* MongoDB Java Driver (mongodb-driver-sync) upgraded from 4.5.1 to 5.5.2

Notable changes:

* `BsonDecimal128.isNumber()` now returns `true` and `BsonDecimal128.asNumber()` returns the equivalent `BsonNumber` (since v5.0).
* `ClusterSettings` computes `ClusterConnectionMode` consistently using the replica set name regardless of how it is configured (since v5.0).
* Dropped server support for MongoDB 3.4 and earlier (since v4.8), MongoDB 3.6 (since v5.2), and MongoDB 4.0 (since v5.5).

MongoDB Java Driver changelog with breaking changes:

* https://www.mongodb.com/docs/drivers/java/sync/current/upgrade/[Full upgrade guide]
* https://github.com/mongodb/mongo-java-driver/releases/tag/r5.0.0[v5.0.0]
** `SocketSettings` timeout parameters changed from `int` to `long`
** `BsonDecimal128` now behaves as a number
** `ClusterConnectionMode` computation changed
** Removed deprecated `ServerAddress.getSocketAddress()` / `getSocketAddresses()`
** Removed `Parameterizable` interface
** Removed `MapCodec` and `IterableCodec` classes
** Removed `StreamFactoryFactory` and related classes
* https://github.com/mongodb/mongo-java-driver/releases/tag/r5.5.0[v5.5.0]
** Dropped support for MongoDB 4.0
** BSON codec lookup and decoding performance optimizations

==== Migration

* Address the MongoDB Java Driver changelog breaking changes listed above. Other migration steps aren’t needed.
