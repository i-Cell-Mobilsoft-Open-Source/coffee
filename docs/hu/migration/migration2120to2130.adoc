= v2.12.0 → v2.13.0

coff:ee v2.12.0 -> v2.13.0 migrációs leírás, újdonságok, változások leírása

=== coffee-module-redis

* Jedis upgradelve 5.1.2-ről  7.2.1-re
* A deprecated `JedisPooled` és `JedisCluster` osztályok lecserélve

Említésre méltó hibajavítások:

* Biztosítva lett a záró kapcsolatot a Pipeline-ban, még akkor is, ha hiba van a szinkronizálási metódusban. https://github.com/redis/jedis/pull/3865[#3865]

Jedis változásnapló a visszafelé nem kompatibilis módosításokkal:

* https://github.com/redis/jedis/releases/tag/v5.2.0[v5.2.0]
** JedisConnectionException contains HostAndPort from DefaultJedisSocketFactory (#3896)
** Address change in JSON.GET command without path (#3858)
** Modify and fail-fast GeoSearchParam (#3827)
** Support transaction from UnifiedJedis without calling multi first (#3804)
** Reduce the log level of validateObject to WARN (#3750)
* https://github.com/redis/jedis/releases/tag/v6.0.0[v6.0.0]
** Make default client side search dialect to 2 (#4060)
** Remove Graph module support (#4073)
** Change FT.PROFILE to return generic object (#4067)
** Remove Triggers and Functions feature (#3969)
** COMMAND INFO reply contains subcommand detail (#4022)
* https://github.com/redis/jedis/releases/tag/v6.1.0[v6.1.0]
* https://github.com/redis/jedis/releases/tag/v6.2.0[v6.2.0]
* https://github.com/redis/jedis/releases/tag/v7.0.0[v7.0.0]
** Removed deprecated constructors, classes, and JedisSharding (#4311)
** Automatic Failover and Failback API changes (#4317, #4310, #4308, #4306, #4297, #4228)
** MultiCluster* classes and methods have been renamed and improved.
** Users upgrading from v6.x should consult the Automatic Failover and Failback Migration Guide.
** https://redis.github.io/jedis/migration-guides/v6-to-v7/#breaking-changes[More details on breaking changes]
* https://github.com/redis/jedis/releases/tag/v7.1.0[v7.1.0]
* https://github.com/redis/jedis/releases/tag/v7.2.0[v7.2.0]
* https://github.com/redis/jedis/releases/tag/v7.2.1[v7.2.1]

* Bevezetésre került a redis sentinel támogatása a modulban
** kiegészült a `sentinel` és 'master' konfigurációs kulccsal a redis konfiguráció, amelynek értéke egy url vagy egy url lista lehet `$(host):$(port)` formátumban valamint a master neve, példa a használatra:

[source,yaml]
----
coffee:
    redis:
        ...
        auth:
            ...
            sentinel:
              - sample-sandbox1.icellmobilsoft.hu:26379
              - sample-sandbox2.icellmobilsoft.hu:26379
              - sample-sandbox3.icellmobilsoft.hu:26379
            master: stream
----
